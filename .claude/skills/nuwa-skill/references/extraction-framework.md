# 女娲智库 - 视角提炼方法论

> 如何从顶级思想家的著作、言论、案例中提炼出结构化的思维框架？

---

## 核心方法：5层提炼法

### 层级1：人物DNA

首先理解这个思想家的**核心信念系统**。

**关键问题:**
- 他/她看重什么？(价值观)
- 解决问题时的首选工具是什么？(思维方式)
- 经常犯什么错误或误区？(盲点)
- 成功的模式是什么？(核心能力)

**数据来源:**
- 自传、公开演讲、采访
- 代表作品或决策
- 与他人的公开辩论
- 粉丝和批评者的评价

**输出:**
```markdown
## 人物DNA

### 核心信念
1. [信念A] — 在什么情况下适用？
2. [信念B]
3. [信念C]

### 思维工具
1. [工具A] — 何时用
2. [工具B]
3. [工具C]

### 常见盲点
1. [盲点A] — 风险
2. [盲点B]

### 成功基因
1. [能力A] — 为什么重要
2. [能力B]
```

---

### 层级2：核心框架

从他们的著作和实践中提炼**具体的思维框架或方法论**。

**关键问题:**
- 他如何分析问题？
- 他的决策流程是什么？
- 他推崇的最高原则是什么？

**常见框架类型:**

#### 2.1 诊断框架
例：Paul Graham 的「如何判断创意的好坏」
```
Graham 框架:
┌─────────────────────────┐
│ 创意评分 = 赚钱能力 × 有趣程度 × 困难程度 │
└─────────────────────────┘
  ├─ 赚钱能力: 有多少人需要？愿意付多少钱？
  ├─ 有趣程度: 你喜欢吗？能坚持吗？
  └─ 困难程度: 技术难度 vs 竞争格局
```

#### 2.2 决策框架
例：Musk 的「第一性原理决策法」
```
Musk 框架:
1. 归纳到最基本的物理定律
   "火箭能便宜吗？火箭基本是铝+燃料，铝=$2/kg，燃料=$0.3/kg"
2. 对比行业现价格
   "NASA火箭 $60M，SpaceX目标 $6M"
3. 反推如何才能达到
   "可回收、工程化、批量生产"
```

#### 2.3 价值框架
例：Naval 的「如何获得财富自由」
```
Naval 框架:
财富 = 专业 × 杠杆 × 选择
  ├─ 专业: 让你与众不同的硬技能
  ├─ 杠杆: 用他人的努力或工具放大
  └─ 选择: 拥有所有权，而非被雇佣
```

**输出:**
```markdown
## 核心框架

### 框架1: [名称]
**适用场景:** [什么问题时用它]

**框架图:**
[ASCII图或结构化描述]

**使用步骤:**
1. 第一步
2. 第二步
3. 第三步

**案例:** [具体例子]

**局限:** [什么时候不适用]
```

---

### 层级3：具体建议

针对**常见问题类型**，提炼他会给的3-5个建议。

**关键问题:**
- 他在相似的问题上说过什么？
- 他的建议背后的逻辑是什么？
- 这些建议的优先级是什么？

**方法:**
1. 收集他关于这类问题的所有言论
2. 归纳共性，去除具体案例的偶然性
3. 排序优先级（通常是他强调次数最多的优先级最高）
4. 标注每个建议的**适用范围**

**输出:**
```markdown
## 对[问题类型]的建议

**场景:** [什么情况下]

**建议1: [核心建议]**
- 原因: [为什么这样做]
- 实施: [怎样做]
- 风险: [要注意什么]
- 例子: [真实案例]

**建议2: [核心建议]**
...

**优先级:** 建议1 > 建议2 > 建议3
（原因: 他最常强调的）
```

---

### 层级4：案例库

收集**他或别人基于他思想的成功/失败案例**。

**关键问题:**
- 他的框架在真实世界中怎样表现？
- 哪些假设在实践中破裂了？
- 成功的关键变量是什么？

**案例结构:**
```markdown
## 案例: [案例名称]

**背景:** [问题 + 情境]

**应用:** [如何用他的框架]

**结果:** [成功还是失败]

**关键洞察:**
- 验证了什么
- 推翻了什么
- 需要补充什么条件

**相关性评分:** ⭐⭐⭐⭐⭐
(你的问题和这个案例的相似程度)
```

---

### 层级5：风险和补充

没有完美的思维框架。识别**这个视角的盲点**以及**需要配合的其他视角**。

**关键问题:**
- 他容易忽视什么？
- 他的框架在什么情况下会失效？
- 哪些问题需要与其他视角综合？

**输出:**
```markdown
## 风险和补充

### 常见盲点
1. [盲点1] — 例: [什么情况下会遇到]
2. [盲点2]

### 适用范围
✅ 特别适合: [问题类型A]
⚠️  谨慎应用: [问题类型B]
❌ 不适合: [问题类型C]

### 推荐配套视角
- 当遇到 [X] 时，参考 [视角Y] 的框架
- [视角Y] 的补充观点是 ...
```

## 实践清单

### 提炼一个新视角时：

- [ ] 收集原始资料（书籍、演讲、采访、文章）
- [ ] 提炼人物DNA（5-7个核心信念）
- [ ] 整理出3-5个核心框架
- [ ] 针对5个常见问题类型，各提炼3个建议
- [ ] 收集5-10个真实案例
- [ ] 标注应用范围和风险
- [ ] 生成对应的 SKILL.md 格式

### 文件结构参考：

```
perspective-skill/
├── SKILL.md              # perspective 的核心内容（用户会读这个）
├── references/
│   ├── bio.md           # 人物简历
│   ├── frameworks.md    # 核心框架详解
│   ├── cases.md         # 案例库
│   └── source-list.md   # 参考文献
└── examples/            # 可选的补充示例
    └── [具体问题案例]/
```

## 示例：Paul Graham 的提炼过程

**第一步：人物DNA**
```
Paul Graham:
- 信念1: "创业 = 帮人解决问题并获得报酬"
- 信念2: "增长是衡量好坏的唯一指标"
- 信念3: "做用户喜欢的事，做到他们想要的程度"
- 工具1: 梯度下降法思维（小步快走）
- 工具2: 对比分析（你的实现 vs 竞争对手）
```

**第二步：核心框架**
```
Graham 框架：
  增长 = 获取用户 × 留存 × 变现
  
  最常见的错误：
  - 过度关注获取，忽视留存
  - 留存太差而硬着头皮做变现
  - 没有聚焦（想做所有事）
```

**第三步：对「如何创业」的建议**
```
1. 找到一个小但真实的问题
   - 例: Y Combinator 的创始人都是这样做的
   
2. 做出最小可行产品（MVP），让用户试用
   - 原因: 反馈 > 计划
   
3. 保持增长优先，其他都是配角
   - 没有增长，再完美的产品也死
```

**第四步：案例**
```
案例: Airbnb
- 初始问题: 年轻创始人，缺乏资金
- Graham 的建议会是: 
  1. 先做出最小版本（拍照上传）
  2. 聚焦一个城市（纽约）
  3. 不断优化留存（改进照片、客户服务）
- 实际步骤: 创始人出去亲自拍照，做用户访谈
- 验证: ✅ 与 Graham 框架相符
```

**第五步：风险**
```
- 盲点: 对创始人心理健康的关注不足
- 对特定行业的理解: Graham 对消费者互联网最有见解，
  对 B2B 工业应用的理解相对较浅
- 推荐配套: 当涉及硬件成本时，参考 Musk 的第一性原理
```

## 常见陷阱

**陷阱1：混淆观点和方法**
❌ "Graham 说要增长，所以增长就最重要"  
✅ "Graham 的框架强调增长作为指标，但前提是有真实用户需求"

**陷阱2：过度泛化**
❌ "Musk 用第一性原理，所以所有决策都用它"  
✅ "第一性原理对物理约束强的领域（火箭、汽车）特别有效，对人文社科领域效果较弱"

**陷阱3：忽视context差异**
❌ "Musk 在2010年做的策略，现在也适用"  
✅ "同样的框架，但市场环境变了，参数需要重新调整"
