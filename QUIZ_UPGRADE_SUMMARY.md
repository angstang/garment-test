### 2. **选项打乱排序** 🔀

**功能**: 每次打开测试，同一道题的选项顺序都会改变

#### 工作原理：
1. **题目随机** (已有): Fisher-Yates算法打乱题目顺序
2. **选项随机** (新增): Fisher-Yates算法打乱每道题的选项顺序
3. **答案追踪**: 系统自动追踪正确答案在新位置的索引

#### 技术细节：
```javascript
// 为每个选项创建 (文本, 原始位置) 对
// 打乱选项位置
// 更新正确答案的新索引
```

**用户体验**:
```
第一次打开:  A. 棉, B. 麻, C. 涤纶, D. 羊毛  (答案: A)
第二次打开:  A. 羊毛, B. 涤纶, C. 棉, D. 麻   (答案: C - 自动更新)
第三次打开:  A. 麻, B. 羊毛, C. 涤纶, D. 棉   (答案: D - 再次更新)
```

### 3. **多选题支持** 🎯

#### 特点：
- 使用**复选框** (checkbox) 而非单选按钮
- 可选择**多个正确答案**
- 必须**全部选对**才算正确

#### 示例：
```
题目: 校服质量检验应包括哪些内容？

☑ 面料检验        ← 正确
☑ 里料检验        ← 正确
☑ 辅料检验        ← 正确
☐ 仅成品检验      ← 不选（这是错的）

⚠️ 系统提示: "这是多选题，可以选择多个正确答案"
```

#### 评分逻辑：
```
用户选择: [面料, 里料, 辅料]
正确答案: [面料, 里料, 辅料]
结果: ✅ 正确

用户选择: [面料, 里料]  (少选了辅料)
正确答案: [面料, 里料, 辅料]
结果: ❌ 错误
```

### 5. **智能评分系统** 🏆

#### 评分规则：

**单选题**：
```
完全正确 = 1分
答案错误 = 0分
```

**多选题**：
```
完全正确 (所有答案都对) = 1分
部分错误或漏选 = 0分
```

**问答题**：
```
答案≥50字 = 1分 (系统自动评)
答案<50字或未作答 = 0分
```

#### 示例计算：
```
总题数: 36道
用户答对:
  - 单选题: 20/24 = 20分
  - 多选题: 4/5 = 4分
  - 问答题: 3/3 = 3分 (因为都≥50字)

总分: 27/36 = 75%
评价: 👍 良好 - 对安全标准掌握较好
```

## 🔧 技术实现细节

### 题型判断

```javascript
question.type === 'single'    // 单选题
question.type === 'multiple'  // 多选题
question.type === 'essay'     // 问答题
```

### 选项打乱实现
```javascript
// 1. 创建选项索引对
optionsWithIndex = [
  { text: "选项A", originalIndex: 0 },
  { text: "选项B", originalIndex: 1 },
  { text: "选项C", originalIndex: 2 },
  { text: "选项D", originalIndex: 3 }
];

// 2. Fisher-Yates打乱
for (let i = arr.length - 1; i > 0; i--) {
  const j = Math.floor(Math.random() * (i + 1));
  [arr[i], arr[j]] = [arr[j], arr[i]];
}

// 3. 更新正确答案索引
newCorrectIndex = optionsWithIndex.findIndex(
  opt => opt.originalIndex === originalCorrectIndex
);
```

### 多选题答案比较
```javascript
const userAnswer = [0, 2];      // 用户选择
const correctAnswer = [0, 1, 2]; // 正确答案

// 比较前先排序（顺序不重要）
const userSorted = [...userAnswer].sort();
const correctSorted = [...correctAnswer].sort();

const isCorrect = JSON.stringify(userSorted) ===
```

## 📞 常见问题

### Q: 问答题没有自动批改？
A: 正确。问答题由系统自动判断是否回答（≥50字），具体内容的正确性显示参考答案供用户自评。

### Q: 多选题为什么要全部选对？
A: 这更符合实际工作要求 - 检验时不能漏掉任何项目。

### Q: 为什么每次选项顺序不一样？
A: 防止用户死记答案。真实工作中需要理解，而不是靠位置记忆。

### Q: 如何在线测试这些功能？
A: 访问 https://angstang.github.io/garment-test/garment_test_v2.html，如果看不到新功能，请强制刷新 (Ctrl+Shift+R)。
